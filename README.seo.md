# SEO Improvements — sivanaltar.com

A log of all SEO work done on the site.

---

## Blog Posts as SEO Landing Pages

Each blog post now has its own SEO-friendly URL and full meta tags.

**URL structure:** `/he/BlogPost/english-slug-here` (slug always in English, regardless of post language)

**DB fields added:**
- `seo_url TEXT` — hyphenated English slug (e.g. `i-started-growing-roots`)
- `keywords TEXT` — comma-separated SEO keywords, generated by Claude from the post content

**Keywords generation:** A migration uses Claude Haiku to read the post title and content and extract 5–10 meaningful SEO keywords per post, in the post's own language.

**Composite unique index:** `(seo_url, lang)` — allows the same slug for both `he` and `en` variants of the same post.

---

## Per-Post Meta Tags

Each blog post page sets the following tags dynamically based on post content:

- `<title>` — post title + site name
- `<meta name="description">` — post summary or truncated content
- `<meta name="keywords">` — post keywords from DB
- `og:title`, `og:description`, `og:url`, `og:image`, `og:type` (`article`)
- `twitter:title`, `twitter:description`, `twitter:image`
- `<link rel="canonical">` — per-post canonical URL
- `hreflang` alternate links for `he` and `en` versions

## Pre-render for Crawlers

Since the site is a React SPA, crawlers (Bing, Facebook, WhatsApp) see empty HTML.

**Solution:** After every deploy, a Playwright script (`scripts/prerender.mjs`) crawls all sitemap URLs and saves full HTML to `dist/prerender/`.

**Nginx** serves pre-rendered HTML first:
```nginx
try_files /prerender$uri/index.html /prerender$uri.html $uri $uri/ /index.html;
```

This means Facebook/WhatsApp/Bing get fully-rendered HTML with correct meta tags.

**GitHub Actions** runs prerender after deploy and rsyncs the output to the server.

## Bing Indexing Fix

`index.html` had a hardcoded `<link rel="canonical" href="https://www.sivanaltar.com/" />` — every page told Bing its canonical was the root, so only the homepage was indexed.

**Fix:** Removed the hardcoded canonical from `index.html`. Canonical is now set dynamically per-page by the React app.
